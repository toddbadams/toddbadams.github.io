Sequential Search

In a sequential search each member of the array is accessed sequentially starting at index 0 and moving toward index N-1, where N is the length of the array. If the item is found it is returned and the search is stopped.  If the end of the array is found (index > N-1) or the value of the current index exceeds the item under search then search ends and -1 is returned, which indicates the item does not exist in the array.

Example Array To Search – [1, 3, 4, 5, 11, 15, 18, 21, 24, 29, 36, 44, 48, 53, 91]

Search Algorithm

Search(A[], Key, Length)
For i = 0 to Length
If (A [i]==Key) return i
If (A [i]>Key) return -1
Return -1

Complexity

Best Case -> O(1) – In the best case the key is found on the first attempt.
Average -> O(N) – On average the key is found after N/2 iterations.
Worst -> O(N) – In the worst case the last iteration contains the key.
